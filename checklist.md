## Блок 2: Каталог и архитектура

### Экзамен

Довести экран каталога до продакшн-качества. Реализовать точное соответствие макету, добавить состояния загрузки и ошибки. Приложение корректно переживает поворот экрана и убийство процесса.

**Сборка**
* Проект собирается без ошибок

**Данные**
* Данные загружаются из JSON-файла [products.json](products.json) через Repository
* Категория «Новинки» формируется на клиенте: товары с тегом `"New"` из всех категорий
* Категория «Новинки» отображается первой в списке табов

**Каталог**
* Табы категорий — переключение фильтрует товары по выбранной категории
* Карточка товара соответствует макету: отступы, шрифты, цвета, скругления
* Карточка товара отображает:
  * Изображение (`imageUrl`)
  * Название (`name`)
  * Цена (`priceInKopecks`) — отформатирована в рублях с разделителями разрядов и символом ₽
* Индикатор загрузки при старте (прогрессбар / skeleton / shimmer)
* Состояние ошибки с возможностью повторить загрузку

**Навигация**
* Нижнее меню с двумя пунктами: каталог и корзина

**Архитектура**
* Данные загружаются через Repository (не напрямую в ViewModel)

**Lifecycle**
* Поворот экрана не крашит приложение и не сбрасывает состояние
* Убийство процесса → восстановление состояния
