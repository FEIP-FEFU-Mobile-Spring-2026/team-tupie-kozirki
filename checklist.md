## Блок 4: API и оффлайн

### Экзамен

Подключить реальный API и реализовать оффлайн-режим. Каталог кэшируется в локальную БД — приложение работает и при отсутствии сети.

**Сборка**
* Проект собирается без ошибок

**API**
* Данные каталога загружаются из API через Repository
* Endpoint: `GET https://fefu2026spring.deploy.feip.dev/catalog`
* Авторизация: заголовок `Authorization: Bearer <token>` (токен — `Cmt7wdwFgDIi1_SRX8hlJIExs0jJKPr4axflLpExAxM`)
* Формат ответа совпадает со структурой [products.json](../hw2/products.json) из второго блока
* Состояние загрузки и состояние ошибки с возможностью повторить запрос

**Оффлайн**
* Каталог кэшируется в БД (Room / CoreData / Hive / Drift / аналог) — не SharedPreferences и не файл с JSON
* Стратегия cache-first: при открытии каталога сначала показывается кэш, параллельно идёт запрос к API; при успешном ответе кэш и UI обновляются
* При отсутствии сети отображаются закэшированные данные без состояния ошибки
* Баннер / snackbar «Нет сети» при отсутствии соединения
* Если кэш пустой и сети нет — корректное состояние ошибки с возможностью повторить
